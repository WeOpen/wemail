# 🧠 use-cases

应用用例层。

## ✅ 放什么
- 明确的业务用例函数
- 路由层调用的流程编排
- 需要组合多个 store / service 才能完成的应用级操作
- 面向“用户动作”或“系统动作”的函数命名

## 🚫 不放什么
- 路由注册与 HTTP 细节
- D1 / R2 的底层实现
- 无业务语义的通用工具函数
- 仅做 re-export 的中转文件

## 📏 边界要求
- route 负责鉴权、取参、调用 use-case、返回响应
- use-case 负责业务规则、权限后的流程控制、审计落点
- service 负责可复用规则辅助，但不替代 use-case 的业务编排
