# ADR 目录

本目录用于存放正式的 Architecture Decision Record（架构决策记录）。

## 放什么

- 已落地的重要架构决策
- 目录结构变更决策
- 技术选型决策
- 测试与发布策略调整决策

## 不放什么

- 临时草稿
- 未决定的 brainstorming 记录
- 与代码实现细节完全重复的文档

## 命名建议

```text
0001-layered-architecture.md
0002-testing-strategy.md
```

## 推荐流程

1. 复制 `docs/adr-template.md`
2. 形成编号
3. 说明背景 / 决策 / 备选方案 / 影响 / 后续动作
