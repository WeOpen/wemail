# Code Review Checklist

## 1. 结构
- 目录职责是否清晰
- 新代码是否放在正确分层
- 是否出现反向依赖

## 2. 代码质量
- 命名是否语义清晰
- 是否存在重复逻辑
- 是否引入了过早抽象
- 注释是否解释“为什么”而不是“做什么”

## 3. 安全与边界
- session / api key / admin 权限边界是否清晰
- 外部输入是否经过 schema / validator 处理
- 错误响应是否明确

## 4. 测试
- 是否新增或更新必要测试
- 是否覆盖关键回归路径
- 验证命令是否真实执行并通过

## 5. 文档
- 目录 README 是否同步
- docs 是否与实现一致
