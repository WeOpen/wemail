# WeMail Personal Settings Redesign

**Date:** 2026-04-17

## Goal

把 `/system/profile` 从占位页升级成苹果式单栏个人设置界面，并且第一版就支持直接编辑关键资料与偏好项。

## Decisions

- 顶部标题栏系统设置二级菜单保持不变：`外观设置` / `个人设置`。
- 页面主体不使用 hero，不做左右分栏；采用单列设置卡片，从上到下按优先级排列：`账号资料 -> 使用偏好 -> 安全与会话`。
- 第一版按“可直接编辑”实施：显示名、语言、时区、日期格式、默认进入页、阅读密度等都直接提供交互控件。
- 缺少后端支持的能力（如真正的密码修改、跨设备会话管理）先做清晰的 UI 承载与按钮语义，但不伪造成功行为。
- 风格参考 Apple 偏好设置：分组清晰、控件克制、说明精简，同时继续使用 WeMail 的浅暖玻璃感和品牌橙强调。

## Page Structure

### 1. 账号资料
- 用户头像占位 / 头像上传入口
- 显示名（可编辑）
- 个人简介 / 备注（可编辑，可选）
- 邮箱（只读）
- 角色（只读）
- 账户创建时间（只读）
- 卡片底部保存按钮

### 2. 使用偏好
- 语言（下拉）
- 时区（下拉）
- 日期格式（分段选择）
- 默认进入页（下拉）
- 邮件阅读密度（舒展 / 紧凑）
- 卡片底部保存按钮

### 3. 安全与会话
- 当前登录邮箱
- 当前会话说明 / 最近活跃时间
- 修改密码按钮
- 退出当前设备按钮
- 退出其他设备按钮
- 对尚未接入后端的动作使用说明性按钮或禁用态，不虚构完成提示

## Interaction Notes

- 每张卡片独立保存，避免整页大表单。
- 修改后显示轻量“未保存”状态。
- 个人设置页中的输入项优先考虑真实本地状态驱动；若尚无服务端接口，先做组件与状态容器，不误导用户为已持久化。
