# WeMail API 密钥页面布局改版设计稿

日期：2026-04-19

## 目标

对 `/api-keys` 页面进行一次布局重组，使其从“左主内容 + 右辅助说明栏”的结构，调整为更偏操作型、更接近开发者后台的页面。

本次改版目标明确为：

- 顶部先展示关键状态
- 中部聚焦“创建 / 管理密钥”和“如何立即使用密钥”
- 去掉会分散注意力的辅助说明模块

## 设计结论

### 1. 页面定位

- 页面类型：开发者凭证管理页
- 第一目标：创建和管理 API 密钥
- 第二目标：拿到密钥后立刻知道如何使用
- 页面风格：**操作型页面**，不是说明型页面

### 2. 布局结构

页面采用两段式结构：

#### 第一层：顶部状态卡

把当前右侧“当前状态”模块拆成 **4 张同级状态卡**，横向平铺在页面顶部。

顺序固定为：

1. 总密钥
2. 活跃密钥
3. 从未使用
4. 已吊销

这些卡片都属于同一个信息层级，不再放在右侧边栏中，也不再使用大段解释文案。

#### 第二层：主内容双栏

顶部状态卡下面，页面改为左右双栏：

- 左侧：个人凭证
- 右侧：快速开始

其中：

- 左侧负责“创建、展示、管理密钥”
- 右侧负责“拿到密钥后如何立即使用”

### 3. 顶部状态卡设计

每张卡统一采用轻量统计卡结构：

- 小标题
- 大数字
- 一行辅助说明

#### 卡片一：总密钥

- 数值：全部密钥数量
- 说明：当前已创建的总凭证数

#### 卡片二：活跃密钥

- 数值：未吊销的密钥数量
- 说明：当前仍可使用的凭证

#### 卡片三：从未使用

- 数值：未吊销且 `lastUsedAt` 为空的数量
- 说明：已创建但尚未接入调用

#### 卡片四：已吊销

- 数值：已吊销数量
- 说明：已失效，不可继续使用

### 4. 左侧：个人凭证

左侧仍然是页面主操作区，保留现有核心能力，但成为更明确的主列。

内部结构顺序：

1. 标题 / 描述 / 创建密钥按钮
2. 创建表单（展开时）
3. 新密钥一次性展示区（创建成功后）
4. 密钥列表或空状态

#### 保留内容

- `个人凭证`
- `API 密钥`
- 创建密钥按钮
- 创建流程表单
- 新密钥一次性展示卡
- 密钥列表
- 空状态
- 吊销按钮
- 复制示例按钮

#### 设计原则

- 左侧永远是主操作区
- 不弱化创建和管理密钥的优先级
- 即使快速开始被放到右边，也不能让右边抢主操作焦点

### 5. 右侧：快速开始

右侧只保留 `快速开始`，不再承担其他辅助说明职责。

内部顺序：

1. 标题 + 一句说明
2. Authorization Header
3. curl 示例
4. 复制代码按钮

#### 设计原则

- 右侧的职责只有一个：
  **拿到密钥后，立刻知道怎么用**
- 不再混入“如何选择接入方式”这种导航型说明
- 不再承担安全策略教育模块

### 6. 删除内容

本次改版明确去掉以下模块：

#### 删除一：安全建议

原因：

- 它会分散页面注意力
- 当前页面更需要聚焦“密钥管理 + 快速使用”
- 安全建议后续可以放到文档或更轻量的位置，而不是占据主页面结构

#### 删除二：接入方式

原因：

- 当前页面目标已经很单纯：管理 API 密钥
- “主动调 API / Webhook / Telegram”的选择说明属于更泛的接入教育内容
- 放在这个页面会打断用户完成 API 密钥操作的节奏

### 7. 状态同步逻辑

#### 无密钥时

- 顶部 4 张状态卡全部正常显示，数值为 `0`
- 左侧显示“创建你的第一个 API 密钥”空状态
- 右侧快速开始仍保留，但使用默认示例 token

#### 有密钥时

- 顶部 4 张卡根据真实列表计算
- 左侧显示列表
- 右侧显示快速开始示例

#### 新建成功时

- 左侧显示一次性 secret 展示卡
- 右侧 `Authorization Header` 与 `curl` 示例同步使用新 secret

#### 吊销后

- 顶部 4 张状态卡实时更新
- 左侧列表状态更新为已吊销
- 右侧快速开始仍然跟随当前 revealState 或默认示例值

### 8. 响应式设计

#### 桌面端

- 顶部 4 卡横向一排
- 下方左右双栏：左个人凭证，右快速开始

#### 中等屏幕

- 顶部 4 卡允许变成 `2 x 2`
- 下方仍保留双栏，但右栏可以更窄

#### 移动端

- 顶部 4 卡改成纵向堆叠或 `2 x 2`
- 下方改成单列：
  1. 个人凭证
  2. 快速开始

### 9. 视觉重点

页面的视觉焦点应调整为：

1. 第一眼：顶部 4 张状态卡
2. 第二眼：左侧创建 / 管理密钥
3. 第三眼：右侧复制示例立即使用

也就是说，这页将从“说明型页面”转为“状态总览 + 主操作区 + 即时上手区”的结构。

## 实施边界

本次只处理 API 密钥页面布局与信息层级，不处理：

- API 密钥真实后端能力扩展
- Webhook / Telegram 页面联动改版
- 新的安全策略后端校验逻辑
- 新的接入方式导航系统

## 测试重点

实现阶段优先验证：

1. 顶部 4 张状态卡是否正确渲染
2. 快速开始是否移动到个人凭证右侧
3. 安全建议与接入方式是否已从页面移除
4. 无密钥 / 有密钥 / 创建成功 / 吊销后的状态统计是否正确
5. 移动端下双栏是否正确降级为单列
