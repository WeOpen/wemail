# WeMail 公告看板设计稿

日期：2026-04-16

## 目标

为 `/announcements` 设计一个**正式后台型**公告看板页面。第一版以 **mock 数据** 为主，兼顾管理员与普通成员阅读场景，优先突出置顶公告，并提供完整控制条的 UI 骨架。

## 设计结论

### 1. 页面定位

- 页面类型：公告看板
- 第一使用者：管理员与普通成员共用
- 第一视觉重点：**置顶公告**
- 公告类型：混合型
  - 系统维护
  - 产品更新
  - 运营通知

### 2. 页面结构

公告看板采用四段式结构：

1. **完整控制条**
   - 搜索框
   - 类型筛选
   - 状态筛选
   - 时间筛选

2. **置顶公告主卡**
   - 作为第一视觉中心
   - 展示当前最重要的一条公告

3. **左侧主时间线**
   - 最近公告列表
   - 按最近更新时间排序

4. **右侧辅助区**
   - 状态概览
   - 近期维护窗口

### 3. 公告字段

每条公告统一使用以下字段：

#### 基础字段

- `id`
- `title`
- `summary`
- `type`
  - 维护通知
  - 产品更新
  - 运营通知
- `status`
  - 进行中
  - 即将开始
  - 已结束
  - 已归档
- `audience`
  - 全部成员
  - 管理员
  - 运营成员
- `priority`
  - 高
  - 中
  - 低
- `publishedAt`
- `updatedAt`
- `author`

#### 可选字段

- `startAt`
- `endAt`
- `tags`
- `pinned`
- `ctaLabel`
- `ctaHref`

### 4. 各区域展示规则

#### 置顶公告主卡

展示：

- 标题
- 2~3 行摘要
- 类型标签
- 状态标签
- 发布时间
- 发布者
- 生效时间范围（如存在）

#### 最近公告时间线

每条卡片展示：

- 类型标签
- 状态标签
- 标题
- 一行摘要
- 范围（谁可见）
- 优先级
- 标签
- 更新时间

#### 状态概览

展示聚合统计：

- 进行中多少条
- 即将开始多少条
- 本周已结束多少条
- 已归档多少条

#### 近期维护窗口

展示：

- 时间范围
- 维护主题
- 一句影响描述

### 5. mock 数据策略

第一版 mock 数据建议包含：

1. **置顶维护通知**
   - 平台升级
   - 数据库维护
   - 回调链路切换

2. **产品更新**
   - 新功能上线
   - 界面改版
   - 策略调整

3. **运营通知**
   - 邀请码规则变更
   - 配额政策变化
   - 使用提醒

目标：让页面看起来像真实后台，而不是单一类型公告堆砌。

### 6. 组件拆分

公告页不再使用 `WorkspacePlaceholderPage`，而是独立页面：

- `AnnouncementsPage`
- `announcementsMockData.ts`
- `AnnouncementsControlBar`
- `AnnouncementsHeroCard`
- `AnnouncementsTimeline`
- `AnnouncementsStatusPanel`
- `AnnouncementsMaintenancePanel`

原则：

- 第一版优先完成真实 UI 壳子
- 筛选条先做静态 UI 与 mock 选中态
- 后续真实接口接入时优先替换数据层和筛选逻辑

### 7. 视觉风格

- 风格：正式后台型
- 置顶公告主卡需要明确，但不能像营销页 banner
- 时间线列表要强调后台内容卡可读性
- 右侧辅助区保持轻量，不抢左侧主列表

### 8. 技术策略

第一版不增加新依赖：

- 控制条、主卡、列表、辅助区：React + CSS
- 状态概览：普通统计卡
- 维护窗口：列表卡片

原因：

- 页面是信息编排问题，不需要图表库
- 第一版目标是结构和阅读体验，不是复杂交互

### 9. 实现边界

本次只做：

- `/announcements` 真页面替换占位页
- mock 数据公告看板
- 完整控制条 UI
- 置顶公告主卡
- 最近公告列表
- 状态概览
- 近期维护窗口

本次不做：

- 真正筛选逻辑
- 发布公告表单
- 公告详情页
- 富文本内容
- 权限操作按钮
- 已读/未读状态

### 10. 验证建议

至少补以下验证：

- `/announcements` 不再渲染占位页
- 控制条出现
- 置顶公告主卡出现
- 最近公告时间线出现
- 状态概览出现
- 近期维护窗口出现
- e2e smoke 可访问 `/announcements`
- 左侧导航和顶部标题行为保持正常

## 结论

第一版公告页应当实现为：

- 正式后台型公告看板
- 置顶公告优先
- 完整控制条
- 左主右辅结构
- 使用 mock 数据
- 不接接口，不做真实筛选
