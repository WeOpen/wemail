# WeMail 管理后台仪表盘设计稿

日期：2026-04-16

## 目标

为管理后台的 `/dashboard` 设计一个以**平台管理员**为第一使用者的仪表盘首页。第一版先使用 **mock 数据**，重点完成信息层级、视觉结构和可扩展的组件边界，不接真实后端接口。

## 设计结论

### 1. 整体方向

- 页面定位：平台管理员首页
- 视觉定位：**大盘展示型**
- 信息策略：**混合型**
  - 同时展示运营数据（收件、发件、活跃度）
  - 同时展示管理数据（用户、邮箱、功能开关、邀请码、配额）
- 暂不包含：
  - 告警区
  - 待办区
  - 实时活动流

### 2. 页面结构

仪表盘采用三段式结构：

1. **顶部 KPI 卡片区**
   - 今日收件
   - 今日发件
   - 活跃邮箱
   - 平台用户
   - 功能开关状态

2. **中部主内容区**
   - 左侧：近 7 天收件 / 发件趋势主图
   - 右侧上：邮箱状态分布
   - 右侧下：用户角色结构

3. **底部补充区**
   - 左侧：账号 / 邮箱增长
   - 右侧：重点资源概览
     - 可用邀请码
     - 默认配额池
     - 最近新增账号

### 3. 数据卡片与图表口径

#### KPI 卡片

- **今日收件**
  - mock：`12,480`
  - 辅助文案：较昨日 `+8.4%`
- **今日发件**
  - mock：`1,284`
  - 辅助文案：成功率 `97.8%`
- **活跃邮箱**
  - mock：`326`
  - 辅助文案：占总邮箱 `74%`
- **平台用户**
  - mock：`89`
  - 辅助文案：管理员 `12` / 成员 `77`
- **功能开关**
  - mock：`3 / 4`
  - 辅助文案：AI、Telegram、外发启用

#### 主趋势图

- 图表类型：双折线
- 主题：近 7 天收件 / 发件趋势
- 配色：
  - 收件量：橙色
  - 发件量：深色
- 用途：作为页面第一视觉中心，优先体现平台活跃走势

#### 结构图

- **邮箱状态分布**
  - 图表类型：环形图
  - 维度：活跃 / 待分配 / 暂停 / 测试
- **用户角色结构**
  - 表达方式：列表 + 占比
  - 理由：后台可读性优先，避免双环图带来的视觉竞争

#### 底部补充图表

- **账号 / 邮箱增长**
  - 图表类型：柱状图
  - 用途：补充平台增长趋势
- **重点资源概览**
  - 表达方式：轻量表格卡片
  - 字段：邀请码、配额池、最近新增账号

### 4. 组件拆分

仪表盘不再沿用当前占位页结构，而是拆成一个真实页面：

- `DashboardPage`
- `dashboardMockData.ts`
- `DashboardKpiGrid`
- `DashboardTrendPanel`
- `DashboardMailboxDistribution`
- `DashboardUserRolePanel`
- `DashboardGrowthPanel`
- `DashboardResourcePanel`

约束：

- `/dashboard` 路由保持不变
- 使用独立 mock 数据文件
- 后续真实接口接入时优先替换数据源，不推翻页面结构

### 5. 视觉风格

- 延续现有后台主题体系：
  - 浅暖底色
  - 深色正文
  - 橙色强调
  - 圆角卡片
- 仪表盘应**弱化当前大文案 hero 感**
  - 顶部摘要更短、更紧凑
  - 第一屏尽快进入 KPI 和图表
- 目标是“有大盘感，但仍然像管理员真的会用的后台首页”

### 6. mock 实现策略

第一版不引入新图表依赖：

- KPI 卡片：React + CSS
- 趋势图 / 环形图 / 柱状图：SVG + mock 数据

理由：

- 不增加依赖
- 第一版仅为静态 mock，结构价值大于图表库能力
- 视觉可控，方便后续再决定是否替换为图表库

### 7. 落地边界

本次设计对应的实现范围：

- 替换 `/dashboard` 当前占位页
- 接入静态 mock 数据
- 做出完整仪表盘 UI
- 不接真实后端接口
- 不修改其他业务页面行为

必须保留：

- 左侧一级菜单结构
- 顶部导航规则
  - 有二级菜单时显示二级菜单
  - 无二级菜单时显示当前一级菜单名称
- 主题切换逻辑
- 现有账号、邮件、用户、设置页面跳转

### 8. 测试建议

至少补以下验证：

- `/dashboard` 渲染新的仪表盘，而不是占位页
- KPI 卡片出现
- 主趋势图标题出现
- 邮箱分布、角色结构、资源概览出现
- admin 登录后可正常进入并查看仪表盘
- E2E smoke 能命中 `/dashboard` 的新页面结构

第一版不做：

- 图表像素级截图对比
- 实时刷新
- 时间筛选器
- 多维度交互筛选

## 实施建议

优先顺序：

1. 用新 `DashboardPage` 替换占位页
2. 接入 mock 数据
3. 落 KPI 卡片和主趋势图
4. 落右侧结构图
5. 落底部补充区
6. 补集成测试与 e2e smoke

## 结论

第一版仪表盘应当是：

- **平台管理员导向**
- **大盘展示型**
- **混合型数据卡 + 趋势 + 分布**
- **真实后台气质**
- **mock 数据可直接替换为真实接口**
